# 命名与存储规则

## 通用原则

1. 文件名尽量使用英文、数字和下划线，避免空格。
2. 文档内容可以用中文。
3. 论文和报告文件保留原始 PDF，同时建立 Markdown 笔记。
4. 同一主题的 PDF、笔记、代码和图表要互相链接。
5. 不把所有资料堆在一个文件夹中，必须归入主题。

## 论文命名

```text
Year_FirstAuthor_ShortTitle.pdf
Year_FirstAuthor_ShortTitle.md
```

示例：

```text
2020_Hersbach_ERA5Reanalysis.pdf
2020_Hersbach_ERA5Reanalysis.md
2018_Claverie_HLSOverview.pdf
2018_Claverie_HLSOverview.md
```

## 政策命名

```text
Year_Agency_ShortTitle.pdf
Year_Agency_ShortTitle.md
```

示例：

```text
2021_StateCouncil_CarbonPeakCarbonNeutrality.md
2024_MEE_CarbonMarketRegulation.md
```

## 方法笔记命名

```text
method_topic.md
```

示例：

```text
footprint_weighted_aggregation.md
site_level_cross_validation.md
hls_cloud_masking.md
vpd_calculation.md
```

## 项目文件命名

项目内部按研究流程编号：

```text
01_problem_definition.md
02_data_sources.md
03_processing_pipeline.md
04_modeling_strategy.md
05_results_and_figures.md
06_limitations.md
07_reusable_experience.md
```
