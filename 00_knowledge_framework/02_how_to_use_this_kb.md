# 如何使用这个知识库

## 增加一条新知识的流程

1. 先判断它是理论、认识、数据技术、方法、政策、应用、项目还是岗位表达。
2. 如果来自论文或报告，先把原文件放入 `papers/` 或 `reports/` 对应主题目录。
3. 再写一份笔记，说明它解决什么问题、我能如何复用。
4. 如果它能进入项目，链接到 `07_projects_and_cases/`。
5. 如果它能进入求职表达，再链接到 `08_career_mapping/`。

## 论文进入知识库的标准

一篇论文不是“下载了”就算进入知识库。至少需要完成：

- 文件归档：PDF 或网页信息放入对应 `papers/` 主题目录。
- 元数据记录：作者、年份、题目、期刊、DOI/URL。
- 内容吸收：写清研究问题、数据、方法、发现、局限。
- 体系连接：链接到相关理论、方法、应用或项目。
- 可复用判断：明确它对我的研究或能力建设有什么帮助。

## 政策和报告进入知识库的标准

政策、标准和行业报告需要记录：

- 发布机构和发布时间。
- 文件级别：法律、条例、部门规章、标准、指南、白皮书、报告。
- 与碳监测、遥感、数据、生态、能源或 ESG 的关系。
- 对企业需求、国家需求或社会需求的启发。
- 可转化为哪些应用场景或岗位能力。

## 周期复盘

建议每两周做一次小复盘：

- 本周期新增了哪些论文、政策和方法？
- 哪些知识只是收藏，尚未吸收？
- 哪些内容可以进入我的项目复盘？
- 哪些短板影响科研或求职？

每三个月做一次大复盘：

- 是否需要调整目录结构？
- 我的研究主干是否更清晰？
- 哪些主题已形成体系，哪些还只是散点？
- 是否形成了新的项目、论文、报告或面试表达？
