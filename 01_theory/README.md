# 01 理论层

理论层回答：这个领域的底层逻辑是什么？

它不是资料堆放区，而是我理解遥感、气象、生态碳循环和时空数据科学的根基。任何方法、应用和项目都应该能追溯到某些理论假设。

## 子模块

| 子模块 | 内容 |
|---|---|
| `carbon_cycle/` | 碳循环、碳源/碳汇、NEE/GPP/RECO/NPP |
| `remote_sensing/` | 遥感物理、光谱响应、植被指数、尺度问题 |
| `meteorology_climate/` | 温度、降水、辐射、VPD、土壤水分、极端气候 |
| `ecology/` | 光合作用、呼吸、物候、生态系统功能 |
| `spatiotemporal_science/` | 空间尺度、时间尺度、代表性、空间自相关 |
| `mathematical_foundations/` | 回归、机器学习、深度学习、泛化、验证 |

## 第一轮重点

1. NEE、GPP、RECO 的定义和相互关系。
2. 光、温、水、VPD 对生态系统碳交换的影响。
3. 遥感植被指数与光合作用/生产力的联系和边界。
4. EC 通量塔的观测支持和 footprint 理论。
5. 时空尺度匹配与 support mismatch。
6. 机器学习/深度学习预测、解释和因果推断的边界。

## 与我的主干研究的关系

我的 HLS-气象-碳通量研究并不只是一个模型任务。它依赖三个理论判断：

1. EC 通量观测是动态源区上的空间积分，而不是固定像元标签。
2. HLS 30 m 光学数据能表达局地地表异质性，但也会放大观测支持错配。
3. GPP、NEE、RECO 对光学观测和气象驱动的响应不同，因此模型收益必须按通量分量和机制边界解释。
