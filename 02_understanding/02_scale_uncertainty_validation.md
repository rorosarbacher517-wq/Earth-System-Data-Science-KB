# 尺度、不确定性与验证：我的判断

## 核心判断

生态遥感建模中，最难的不是单纯提高模型精度，而是证明模型学到的东西在正确的尺度上成立，并且能经得起独立验证。

## 三个尺度问题

1. 空间尺度：HLS 像元、EC footprint、ERA5 网格和生态区边界不同。
2. 时间尺度：半小时通量、卫星过境、小时气象、日/月/年汇总不同。
3. 过程尺度：光合作用、呼吸、土壤水分记忆、扰动恢复的响应时间不同。

## 不确定性来源

| 来源 | 表现 |
|---|---|
| 观测误差 | EC 通量噪声、QC、gap filling |
| 通量分解 | GPP/RECO 不是直接观测，而是分解结果 |
| 遥感误差 | 云、BRDF、传感器差异、重建误差 |
| 气象误差 | ERA5 尺度粗、局地气象差异 |
| Footprint 误差 | FFP 假设、输入变量误差 |
| 模型误差 | 过拟合、泄漏、结构偏差 |
| 外推误差 | 新站点、新气候区、新生态类型 |

## 验证原则

1. 不只报告 pooled R2/RMSE，也报告站点级结果。
2. 不只看均值，也看分布和极端失败案例。
3. 不只比较模型，还要比较控制实验。
4. 不只解释变量重要性，还要解释边界条件。
5. 不只写“效果好”，还要写“什么时候有效、什么时候不可靠”。

## 对我成长的意义

如果我能长期把“尺度、验证、不确定性”作为研究习惯，就会比只会调模型的人更有竞争力。这是科研、企业和政策场景都需要的成熟能力。
