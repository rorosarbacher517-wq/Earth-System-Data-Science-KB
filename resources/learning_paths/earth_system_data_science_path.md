# 地球系统数据科学学习路线

## 阶段 1：概念地基

目标：把核心概念弄清楚，不急于上模型。

- NEE、GPP、RECO、NPP。
- 碳源、碳汇、碳循环。
- 遥感反射率、NDVI、EVI、NIRv、SIF。
- 温度、辐射、VPD、土壤水分、降水。
- EC tower、footprint、observation support。

输出：

- 完成核心术语表。
- 每个概念写一页笔记。

## 阶段 2：数据体系

目标：知道每种数据从哪里来、代表什么、有什么限制。

- HLS/Landsat/Sentinel/MODIS。
- ERA5/ERA5-Land。
- FLUXNET/AmeriFlux。
- 土地覆盖、DEM、气候区、站点元数据。

输出：

- 每个数据源写数据卡片。
- 记录访问方式、变量、分辨率、许可和引用方式。

## 阶段 3：方法体系

目标：把数据处理、特征工程、建模和验证串成流程。

- HLS QC。
- 气象特征工程。
- 站点-像元匹配。
- Footprint-weighted aggregation。
- Site-level CV。
- 支持组成诊断。
- 解释性和稳健性分析。

输出：

- 每个方法写方法笔记。
- 每个方法至少连接一个项目案例。

## 阶段 4：机制与边界

目标：从“模型预测”进入“机制解释和证据判断”。

- GPP、NEE、RECO 分量差异。
- 预测、解释、因果的边界。
- 尺度、不确定性和外推风险。
- 生态理论与机器学习诊断的结合。

输出：

- 完成机制判断笔记。
- 梳理项目中最重要的反例和边界条件。

## 阶段 5：应用与知识迁移

目标：把知识迁移到碳监测、MRV、气候风险和生态治理。

- 碳监测与 MRV。
- 生态系统气候风险。
- ESG 环境数据系统。
- 数字能源与碳管理。
- 遥感产品验证。

输出：

- 每个应用方向写一页问题地图。
- 每个应用方向明确数据、方法、边界和可交付结果。
