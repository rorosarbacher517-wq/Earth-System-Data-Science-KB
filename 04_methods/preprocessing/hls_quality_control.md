# HLS 质量控制与有效像元检查

## 解决什么问题

HLS 是光学遥感产品，云、云影、雪、气溶胶、传感器缺测和重建误差都会影响植被指数和模型输入。若不做 QC，模型可能学习到云污染而不是生态过程。

## 输入

- HLS 反射率波段。
- HLS Fmask/质量标记。
- 站点周围 patch。
- 目标日期或合成窗口。

## 输出

- 有效像元 mask。
- 有效像元比例。
- 清洗后的反射率/植被指数。
- QC summary 表。

## 核心步骤

1. 读取 HLS 反射率和质量标记。
2. 屏蔽云、云影、雪、水体或异常值。
3. 检查各波段有效范围。
4. 计算 NDVI、EVI、NIRv 等指标。
5. 统计 patch 内有效像元比例。
6. 若有效像元比例过低，标记为低可信样本或剔除。

## 常见错误

- 把无效值 `-9999` 当成真实数值。
- 对不同传感器未做一致化处理。
- 只看中心像元，不看 footprint 源区内有效像元。
- 不记录剔除比例，导致后续结果不可追踪。

## 与 footprint 的关系

如果 footprint 权重落在大量无效 HLS 像元上，那么 tower-scale aggregation 也会不稳定。因此，HLS QC 不只是遥感预处理问题，也是 observation operator 可信度问题。
