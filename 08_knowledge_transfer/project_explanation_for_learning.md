# HLS-footprint 碳通量项目解释

## 学习目的

这个项目用于理解一个更一般的问题：当地面观测、遥感像元和模型输入具有不同空间支持时，如何设计数据融合和验证流程，使模型学习到的结果更接近真实观测机制。

## 问题背景

EC 通量塔提供半小时尺度碳交换观测，但每条观测的源区会随风向、稳定度和湍流变化。传统遥感上尺度常用固定像元或固定窗口与塔点配对，这隐含了一个假设：塔观测和遥感特征代表同一固定空间范围。

这个假设在空间异质性较强的景观中可能不成立。HLS 30 m 数据提高了对局地地表异质性的表达能力，但也使 source-area 轻微移动带来的组成差异更明显。

## 数据结构

| 数据 | 作用 |
|---|---|
| HLS 30 m 光学数据 | 表征 source-area 内植被状态和地表异质性 |
| ERA5/气象数据 | 提供辐射、温度、VPD、水分等过程驱动 |
| EC 通量数据 | 提供 NEE、GPP、RECO 等目标变量 |
| Footprint 权重 | 表征每个半小时通量观测的动态空间支持 |
| 站点元数据 | 支持分层分析和结果解释 |

## 方法逻辑

```text
HLS patch + meteorology + time
  -> latent pixel-level flux field
  -> footprint observation operator
  -> tower-scale NEE/GPP/RECO prediction
```

Footprint 不只是一个输入特征，而是观测算子：它定义像元级潜在通量场如何聚合到塔尺度观测。

## 关键验证

1. Same-backbone comparison：保持输入、模型架构、训练协议和验证划分一致，只改变空间聚合算子。
2. Site-level CV：同一站点的样本不能同时进入训练和测试。
3. 支持组成诊断：比较动态 footprint 与固定窗口在 NDVI、土地覆盖或异质性指标上的差异。
4. 分量解释：分别讨论 GPP、NEE、RECO 的收益和边界。

## 可迁移经验

- 点位观测不等于点位真值，必须考虑观测支持。
- 高分辨率数据提高信息量，也提高空间错配敏感性。
- 模型提升需要通过控制实验与验证设计来解释。
- 机制解释必须区分预测、相关性、机制一致性和因果。
