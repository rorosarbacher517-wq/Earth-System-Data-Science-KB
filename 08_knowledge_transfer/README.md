# 08 知识迁移层

知识迁移层回答：这套知识如何从一个具体研究问题迁移到更广泛的科学、工程、政策和应用场景？

它的作用是把前面七层知识重新组织成跨领域可理解的能力结构，例如：从 EC 通量塔到遥感产品验证，从 footprint 支持匹配到一般观测支持问题，从碳通量估计到 MRV 和气候风险。

## 主要内容

| 文件 | 作用 |
|---|---|
| `cross_domain_application_map.md` | 将知识体系连接到相邻领域和应用问题 |
| `project_explanation_for_learning.md` | 用学习和研究复盘的方式解释 HLS-footprint 项目 |
| `knowledge_capability_matrix.md` | 梳理知识模块、证据来源、可迁移能力和待补短板 |

## 使用原则

1. 先保持科学问题的准确性，再谈应用。
2. 不把应用价值写成口号，要说明数据、方法、边界和不确定性。
3. 写清“这个知识可以迁移到什么类型的问题”，并说明迁移所需的数据、方法和边界条件。
4. 每个迁移方向都要能回链到理论、数据、方法或项目证据。
