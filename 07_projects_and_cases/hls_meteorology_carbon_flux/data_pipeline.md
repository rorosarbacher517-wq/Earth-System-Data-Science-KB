# 数据流程

## 目标

构建 HLS、气象、EC flux 和 footprint 在站点、日期、半小时尺度上同步的建模样本。

## 输入

1. EC 通量数据：NEE、GPP、RECO、QC flags。
2. HLS 光学数据：反射率、质量标记、植被指数。
3. 气象数据：辐射、温度、VPD、土壤水分、降水等。
4. Footprint 数据：每个半小时的源区权重。
5. 站点元数据：经纬度、IGBP、气候区、测量高度等。

## 核心步骤

1. 通量数据 QC 和目标变量选择。
2. HLS 有效像元检查、云掩膜和时序同步。
3. 气象数据提取、单位转换和半小时/日尺度对齐。
4. Footprint 权重计算或读取，并与 HLS 像元网格匹配。
5. 构建 site-day 样本，保留 48 个半小时 token。
6. 按站点进行 train/test fold 划分。
7. 只用训练 fold 拟合归一化参数，避免数据泄漏。

## 质量控制关注点

- HLS 缺测、云污染和低有效像元比例。
- EC 通量 QC、gap-filled 变量和 partitioning 不确定性。
- ERA5 等再分析数据的空间尺度与站点尺度差异。
- Footprint 权重是否归一化，是否覆盖足够 HLS 有效像元。
- 同一站点样本不能同时进入训练和测试集。
