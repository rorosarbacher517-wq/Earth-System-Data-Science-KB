# 06 应用层

应用层回答：这些知识能解决哪些真实问题？

应用层不是展示包装，而是把理论、数据、方法和政策连接到企业、社会、国家和世界需求。

## 应用方向

| 应用 | 现实问题 | 需要的能力 |
|---|---|---|
| `carbon_monitoring_mrv/` | 如何监测、报告和核查碳排放/碳汇 | 遥感、通量、碳核算、MRV |
| `digital_energy/` | 能源转型和碳管理如何数据化 | 碳数据、预测模型、系统集成 |
| `smart_city_ecology/` | 城市生态和热环境如何监测 | 遥感、GIS、时空分析 |
| `agriculture_remote_sensing/` | 作物长势、产量和碳汇如何估计 | 植被指数、气象、作物模型 |
| `climate_risk/` | 极端气候如何影响生态和资产 | 气象、风险指标、情景分析 |
| `esg_sustainability/` | 企业如何披露和管理环境数据 | ESG、碳核算、数据治理 |

## 应用笔记模板

每个应用方向建议按以下结构整理：

1. 现实问题。
2. 相关理论。
3. 需要的数据。
4. 核心技术。
5. 常用方法。
6. 政策或产业背景。
7. 可交付产品。
8. 相关数据产品或研究问题。
9. 已有基础。
10. 需要补强。
